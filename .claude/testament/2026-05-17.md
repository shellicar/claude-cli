Courier — closing the launch-flags arc.

## What shipped

Four new launch-time flags on `claude-sdk-cli`, plus a documentation fix for an
existing flag:

- `--name <value>` — display label for the session, rendered in bright white with
  a `*` prefix, replacing the cwd-derived label in the status bar.
- `--model <value>` — launch-time model override that beats config and survives
  config-reload, shaped as a mutable slot so issue #309 can later set it from
  command mode without further refactor.
- `--prompt <text>` — send `<text>` as the first message of a fresh session.
  Composable with `--file`: both arrive as a single `UserInput` via the same
  `buildSubmitText(text, attachments)` call the interactive submit path uses.
- `--no-resume` — start fresh instead of resuming the latest session for the cwd,
  with no initial turn.
- `--file` is now listed in `--help` (was implemented but undocumented).

## Design decisions worth keeping

**Single resolver for model reads.** A `getEffectiveModel()` closure in `main.ts`
returns `overrides.model ?? configLoader.config.model`. Every model-read site
goes through it: the `mapConfig` model field and the four `setModel` call sites
(the `onChange` callback, plus the post-startup, pre-turn, and post-turn calls).
Scattered `??` substitutions across those five sites produce a two-form drift
that's easy to miss — one site reads from the callback's fresh `config`
parameter, the others from `configLoader.config`. The resolver collapses both
into one call.

Placement matters: between `configLoader.load()` and `configLoader.onChange(...)`
registration. After `load()` so `configLoader.config` is populated at first
call; before `onChange` so the callback closes over the resolver in lexical
order (closures would resolve a later declaration too, but top-to-bottom reading
is what we want).

**Mutable `overrides` object, not a `const`.** Seeded from the launch flag, but
the object itself is mutable so issue #309 can swap `overrides.model` from a
command-mode callback without touching `mapConfig` or any other read site.

**Deliberate exception that looks like a bug.** The `logger.info('config reloaded',
{ model: config.model })` line keeps `config.model`, *not* `getEffectiveModel()`.
That line records what the loader received from disk — the audit-useful value
when debugging an override situation. Don't "fix" it.

**One construction path for the initial turn.** `buildFileInput` became
`buildInitialInput`, gained a `text: string` parameter, and returns early when
`filePath` is null. The `--prompt` + `--file` combination produces one
`UserInput` through the same call shape as the interactive submit. There's no
parallel construction path to drift against.

**Name display is one field, two sources.** `StatusState` stores `#sessionName`;
`renderModel` in `renderStatus.ts` picks `label = sessionName` (with `*` prefix
and `BOLD_WHITE`) when set, else the cwd basename. `BOLD_WHITE` was already
exported from `@shellicar/claude-core/ansi`, so no new colouring path.

## What to know about this branch

**Behind origin/main by a release cycle.** The branch was cut on 2026-05-11 and
sat dormant for six days. In that window, main shipped a security fix
(GHSA-p7fg-763f-g4gf), package version bumps, regenerated changelogs, and a
dependency refresh that touched `pnpm-lock.yaml` significantly. The
`origin/main..HEAD` diff stat shows ~40 files for that reason, not because this
PR's commit touched them. The PR will need an `origin/main` merge before it can
land cleanly; the Courier protocol handles that at PR-open time.

**Line-number drift in the plan.** The Apostle's ground-truth index had 1–3
line drift from the actual file at the `setModel` sites. It didn't matter
because every edit anchored to surrounding code context (the literal
`statusState.setModel(configLoader.config.model);` substring) rather than to a
line number. If you're walking new code into this file, anchor by context — the
file shifts under you faster than the index updates.

## Tests

Two new describe blocks: `StatusState — sessionName` (2 tests) and
`renderModel — session name override` (4 tests). Both follow the
`expected`/`actual` named-variable convention the `tdd` skill prescribes, which
the existing `renderModel — model abbreviation` block in the same file already
uses. The earlier inline `expect(actual).toContain(...)` form remains in the
`renderModel — empty state` block; the new code didn't unify it because the
mission scope was launch flags, not test-style cleanup.

The resolver itself isn't unit-tested. It's a closure inside `main.ts` and
exposing it would mean restructuring the IIFE, which the brief explicitly
forbade. Every existing model-read test still passes because `overrides.model`
defaults to `null` and the resolver falls through to config.

Total suite: 497 tests across 24 files, all passing.
