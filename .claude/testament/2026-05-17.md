## Carried forward from prior phases

**The `overrides` slot is mutable on purpose.** `main.ts` declares
`const overrides = { model: ... }` (mutable object, not a `const` shape) so
issue #309 can later swap `overrides.model` from a command-mode callback
without touching `mapConfig` or any other read site. If you're picking up
#309, the slot is waiting for you; don't refactor it thinking it should be
`const`.

**Don't "fix" the `config.model` in the config-reload log.** The line
`logger.info('config reloaded', { model: config.model })` in
`configLoader.onChange` deliberately uses `config.model`, not
`getEffectiveModel()` like every other model-read site. The log records what
the loader received from disk, which is the audit-useful value when
debugging an override situation. A future contributor will see this and
think it's a missed conversion. It isn't.

## Operational notes from this phase

**`gh pr edit --add-reviewer` is silent on failure.** First call printed the
PR URL and exit 0; `gh pr view --json reviewRequests` showed `[]` immediately
after. Retried after the merge-push and it landed. Cause unknown — possibly
a timing window around upstream setup or auto-merge enablement. Verify with
the JSON view, don't trust the URL echo.

**`gh pr merge --auto --squash` accepts on a CONFLICTING PR.** Enabled it
while `mergeable: CONFLICTING` and `mergeStateStatus: DIRTY`. Empty stdout,
exit 0; `autoMergeRequest.enabledAt` showed the pre-resolution timestamp
afterward. GitHub queues the auto-merge regardless of current mergeability
and activates once mergeable. Order doesn't matter.

**`git status` right after a local merge can mislead.** After `git merge
origin/main` produced a merge commit, `git status` reported "Your branch is
up to date with 'origin/feature/launch-flags'" even though local was 2
commits ahead. The "up to date" line compares against the cached remote
ref, not actual ahead/behind after a local merge. `git rev-list --count
--left-right` is the reliable check.

**Wide `origin/main..HEAD` diff stat is divergence, not conflict.** Six
days of staleness translated to ~40 files in the stat (security fix,
package bumps, regenerated changelogs, ~1000 lines of lockfile churn).
The merge applied cleanly — only `apps/claude-sdk-cli/changes.jsonl`
needed line-level auto-merge, no conflict markers. A future Courier seeing
the same shape on a stale branch shouldn't pre-emptively assume conflict.
