## Operational notes worth keeping

**`gh pr edit --add-reviewer` is silent on failure.** First call printed the PR
URL and exit 0; `gh pr view --json reviewRequests` showed `[]` immediately
after. Retried after the merge-push and it landed. I don't know the cause ‚Äî
possibly a timing window around branch upstream setup or auto-merge enablement.
Verify with the JSON view, don't trust the URL echo.

**`gh pr merge --auto --squash` accepts on a CONFLICTING PR.** Enabled it while
`mergeable: CONFLICTING` and `mergeStateStatus: DIRTY`. The command returned
empty stdout and exit 0; `autoMergeRequest.enabledAt` showed the pre-resolution
timestamp afterward. GitHub queues the auto-merge regardless of current
mergeability and activates once mergeable. Order doesn't matter ‚Äî enable before
or after resolving the conflict.

**`git status` right after a merge can mislead.** After `git merge origin/main`
produced a merge commit, `git status` reported "Your branch is up to date with
'origin/feature/launch-flags'" ‚Äî but local was 2 commits ahead. The "up to
date" line compares against the locally cached remote ref, not actual ahead/
behind after a local merge. `git rev-list --count --left-right` is the reliable
check.
