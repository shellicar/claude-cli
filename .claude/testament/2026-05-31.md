Courier for launch-flags-followup. Eight items shipped across three operator phases.

## What was built

**Items 1-4 (new flags and config)**

- `statusBar.showConversationId` config key (default `true`). Conversation id appears in the status bar after model and cwd. Controlled by `statusBarSchema` in `src/cli-config/schema.ts`.
- `--resume <conversationId>`: UUID-validated at entry via `z.string().uuid().safeParse`, bypasses the marker-file read, loads from `~/.claude/conversations/{id}.jsonl`. Absent history is a silent no-op (not an error). Implemented via `ConversationSession.resume(id)` which calls the private `#loadHistoryForId(id)` helper. `load()` was refactored to call the same helper, eliminating the duplicated history-load body that previously lived in both places.
- `--file` accepts multiple values. Argv order = attachment order. `initialFilePath` renamed to `initialFilePaths` everywhere. `buildInitialInput` replaced with a loop over the array.
- `--prompt` escape decoding in `src/decodePromptEscapes.ts` (top-level under `src/`, not `model/` or `view/` -- only consumed by `main.ts`). Handles `\n`, `\r`, `\t`, `\\`. Unknown escapes preserved as-is.

**Items 5-6 (parseArgs hardening and visual marker)**

- `parseArgs` is `strict: true`. Unknown flags throw; the catch block prints the error and exits. `-?` is intercepted pre-parseArgs on `process.argv` before the try/catch, so it still works as a help alias.
- `--model` override shows `*` suffix on the model in the status bar (e.g. `Opus 4.6*`). Distinct from `--name` which uses `*` prefix. `setModel(name, overridden = false)` in `StatusState` stores the flag; all call sites in `main.ts` pass `overrides.model != null`.

**Items 7-8 (model name/version split)**

- `parseModelName(modelString)` in `src/view/parseModelName.ts`. Family = first non-numeric token after `claude`. Version = trailing numeric tokens joined by `.`. No trailing numerics = `null`. Old-style names (e.g. `claude-3-5-sonnet-20241022`) are out of scope and not special-cased.
- `renderModel` in `src/view/renderStatus.ts` was fully rewritten to call `parseModelName`. `abbreviateModel` was deleted. Status bar shows `Name 4.6` or just `Name` when no version.

## Key traps

**`as string[]` cast (Item 3).** `ReturnType<typeof parseArgs>` widens `values.file` so `Array.isArray` narrows to `(string | boolean)[]`, not `string[]`. The plan didn't enumerate this consequence. The cast is safe at runtime (`type: 'string', multiple: true` guarantees string elements) but it's there. Future operators touching `values.file` should expect the widened type.

**Two variables named `parsed` in `main.ts`.** `let parsed: ReturnType<typeof parseArgs>` at the parseArgs site, and `const parsed = z.string().uuid().safeParse(resumeId)` inside the UUID check block. They shadow cleanly today. If the UUID check is ever lifted out of its block, the name collision becomes an error.

**IIFE vs guarded block.** An early plan draft used an IIFE for UUID validation (needed branching + a return value). The local idiom is one-liner extraction + guarded block at the same scope. When you see conditional logic that produces one value and needs to exit on bad input, reach for the guarded-block pattern, not a function wrapper.

**Duplication trap.** The Apostle's first draft had `resume()` duplicating the history-load fragment of `load()`. The signal to catch: prose that says a new method "mirrors" or "parallels" an existing one. Mirroring exactly is duplication. The correct shape is a shared private helper called from both.
