## The lifecycle divergence

The `prompt` block is added via `addBlocks()` in `AppLayout.startStreaming`, not via
`transitionBlock`. This bypasses the clock-stamp path. The fix is to route it through
`transitionBlock` + `appendToActive` + `completeActive` in `startStreaming`. Both
timestamps land sub-millisecond apart (duration `0s`), which is accurate — the
prompt is complete the instant it enters the conversation.

History replay and startup banner blocks go through `addBlocks` with no timestamps.
Their dividers render without timestamp fields. This is correct; we can't recover
historical timing.

## @js-joda/core and timezone

The `claude-sdk-cli` package doesn't have `@js-joda/core`. The legacy CLI uses it at
`^5.7.0`. Add at that version for consistency.

`ZoneId.systemDefault()` requires `@js-joda/timezone`. Skip it — use native `Date`
to format `Instant.toEpochMilli()` as local time. Avoids the timezone extension
dependency entirely.

## Clock injection pattern

`ConversationState` takes `clock: Clock = Clock.systemUTC()` as a constructor
parameter. Production code (`AppLayout`) uses the field initializer
`#conversationState = new ConversationState()` unchanged (default clock applies).
Tests inject `Clock.fixed(instant, ZoneId.UTC)` or a `FakeClock extends Clock` for
two-instant scenarios (enter at T1, exit at T2).

## Spread-on-seal

Current code: `this.#sealedBlocks.push(this.#activeBlock)` — pushes the reference.
New code: `this.#sealedBlocks.push({ ...sealing, exitedAt: Instant.now(this.#clock) })`.
The spread creates a new object with `exitedAt` injected. `appendToLastSealed` mutates
the sealed block directly via index — this still works on the spread copy.

## Continuation suppression + timestamps

Consecutive same-type blocks suppress later dividers. Only the first block's divider
renders, carrying that block's `createdAt`/`exitedAt`. The last block in the run
has its own `exitedAt` but its header is suppressed. Accepted without special-casing.

## buildDivider signature

New optional third arg `timestamps?: DividerTimestamps`. Existing callers
(AppLayout.render for the live input area prompt divider, and the null-label
separator) pass no timestamps and are unaffected. The type is exported for test use.

# 02:43 — Plan revision (r2)

SC reviewed and made two corrections.

**1. Time-of-day display: use @js-joda/core, not native Date.**
My premise was wrong: `@js-joda/core` resolves system-default local time on its
own via `ZoneId.systemDefault()`. The `@js-joda/timezone` extension is only
required for named IANA zones. The correct implementation is:

```ts
const TIME_FORMAT = DateTimeFormatter.ofPattern('HH:mm:ss');
function formatInstantToTime(instant: Instant): string {
  return instant.atZone(ZoneId.systemDefault()).toLocalTime().format(TIME_FORMAT);
}
```

The native `Date` fallback was unnecessary and broke the one-library principle.

**2. createdAt comment: the optionality is intentional, document it.**
The field already had `?` (correct), but no explanation. SC wants a code comment
on `createdAt?` in the Block type diff so the next reader doesn't read the
optionality as an accident. Added:
```
/** Set when the block is opened via transitionBlock. Absent for blocks added via
 * addBlocks (history replay, startup banner) where no creation instant is available. */
```

Both changes are surgical. Everything else in the plan is unchanged.
