## Phase 2 complete: Green tests and implementation

Branch: `feature/audit-persistence`. Six files staged, not committed.

### What was implemented

1. **`AnthropicClient.ts`**: Added private `EventEmitter<AnthropicClientEvents>` (composition, not inheritance). Exposed typed `on()` and `off()` methods. In `stream()`, the `BetaMessageStream` returned by `this.#raw.beta.messages.stream()` now has a `finalMessage` listener that re-emits on the private emitter. The emit is wrapped in try/catch so listener errors never propagate into stream processing.

2. **`packages/claude-sdk/src/index.ts`**: Added `BetaMessage` to the type re-export line alongside `BetaMessageParam`.

3. **`MemoryFileSystem.appendFile`**: Gets existing content (or empty string), concatenates, sets. Creates the file if it doesn't exist.

4. **`NodeFileSystem.appendFile`**: Uses `mkdir(dirname(path), { recursive: true })` then `appendFile(path, content, 'utf-8')` from `node:fs/promises`. Same directory-creation pattern as `writeFile`.

5. **`AuditWriter.write()`**: Builds `{ timestamp: new Date().toISOString(), ...msg }`, serializes as single-line JSON with trailing newline, calls `this.#fs.appendFile()`. The promise `.catch()` does `console.error` + `process.exit(1)` (fatal on error, matching the existing audit design).

6. **`main.ts`**: After `AnthropicClient` construction, creates `AuditWriter` with `nodeFs` and `~/.claude/audit` dir, subscribes `client.on('finalMessage', (msg) => auditWriter.write(session.id, msg))`.

### Test results

- **439 tests pass** across 20 spec files (was 434 in Phase 1 with 4 new failing; now 5 AuditWriter tests pass)
- Type-check passes
- Build passes

### Design notes

- `write()` is synchronous in signature (returns `void`) because the caller (the event listener) doesn't await it. The async `appendFile` call is fire-and-forget with a fatal `.catch()`. This matches the prompt's design: fatal on error.
- `NodeFileSystem.appendFile` creates parent directories before appending, same as `writeFile`. This means the audit directory doesn't need to exist beforehand.
- The `BetaMessageStream` from `@anthropic-ai/sdk` (v0.82.0) emits `finalMessage` with the fully assembled `BetaMessage` after the stream completes. This is the natural place to capture the complete response.
- Three unrelated files were modified by `pnpm ci:fix` (biome auto-formatting on pre-existing issues in `reflow.ts`, `Find.ts`, `Find.spec.ts`). These are NOT staged.
