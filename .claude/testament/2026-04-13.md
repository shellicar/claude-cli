# Audit Persistence

## Design

EventEmitter composition on `AnthropicClient` (Design 3). The client holds a private typed `EventEmitter` and exposes `on`/`off`. When `BetaMessageStream` emits `finalMessage`, the client re-emits on its own emitter. The emit is wrapped in try/catch so listener errors never propagate into stream processing.

`AuditWriter` takes `IFileSystem` and an audit directory path. It subscribes to `finalMessage` and writes `{ timestamp, ...msg }` as JSONL via `IFileSystem.appendFile`. Fatal on error (console.error + process.exit(1)).

## Traps

**`BetaMessage` import path**: In `@anthropic-ai/sdk` v0.82.0, `BetaMessage` is not re-exported from `@anthropic-ai/sdk/resources/beta.js`. You have to import from the deep path `@anthropic-ai/sdk/resources/beta/messages/messages.js`. The SDK re-exports it from `@shellicar/claude-sdk` index so consumers don't hit this.

**`BetaMessage` test construction**: Requires `container`, `context_management`, `stop_details` fields that aren't obvious from the type name. The test helper `makeMessage()` uses `as BetaMessage` cast because `BetaUsage` has optional fields that differ across SDK versions.

**Biome import ordering**: Adding `BetaMessage` to an existing re-export line can trigger biome import-sort failures. CI catches this but the error message doesn't make the cause obvious.

## What was added to `IFileSystem`

`appendFile(path: string, content: string): Promise<void>` on the abstract class. `NodeFileSystem` creates parent directories before appending (same pattern as `writeFile`). `MemoryFileSystem` concatenates to existing content or creates.
