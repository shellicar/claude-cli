## Key observations

The `tool_approval_request` tests use `toulu_01` (typo, missing `c`) consistently for both
`streamTool(handler, 'toulu_01', ...)` and `requestId: 'toulu_01'`. This is from the plan —
the typo doesn't break anything because both sides of the map lookup use the same string.
The Apostle flagged it.

The `server_tool_result` existing test "unknown block types do not emit server_tool_result"
uses `processor.on('server_tool_result', (name) => emitted.push(name))` with a single
parameter callback. TypeScript allows this (fewer params assignable to more) even though
the event now emits `(id, name, result)`. The callback captures `id` into `name`, but
since the test asserts length 0 (no events fire), it passes correctly.
