## The schema stub trap

`z.any()` was my first attempt for the schema stub. In Zod v4, `z.any()` without `.optional()` or `.default()` in a `z.object()` causes parsing to fail entirely when the field is absent — breaking all tests that call `parse({})`, not just the `systemPrompt` tests. Switched to a proper structured schema with wrong boolean defaults (`false` instead of `true`). This keeps the structure correct (so `initConfig.spec.ts` passes) while making the defaults/fallback tests fail.

## The unused `systemPromptSourcesSchema`

The schema file declares `systemPromptSourcesSchema` (correct all-true defaults) but the stub's `systemPromptSchema` inlines its own sources schema with false defaults. This is intentional: `systemPromptSourcesSchema` is declared with the correct shape for the Builder to reference, and TypeScript won't error on the unused declaration (no `noUnusedLocals`). The Builder will wire it in.

# 22:34

Builder phase for configurable system prompts (#284). Implemented sections A–G from the Apostle plan against the Scaffolder's stubs.

## What was implemented

**A. NodeDirectoryWatcher** — filled the `watch()` body: groups paths by parent directory, installs one `fs.watch` per directory, filters events by basename. One deviation from the plan: the plan included a `filename.toString()` fallback for `Buffer` filenames. Node's typed callback has `string | null` (not `string | Buffer`) in this overload, so `typeof filename === 'string'` narrowed the else branch to `never` and caused a DTS build error. Removed the fallback; after the null guard, `filename` is already `string`. Deleted `NodeConfigWatcher.ts`.

**B. SystemPromptLoader** — added `resolve` import, `DEFAULT_SOURCES`, `SystemPromptFile` type, `systemPromptFiles()`, `readIfPresent()`, and the full `getSections()` body. Matches the plan exactly.

**C. composeSystemPrompts** — implemented the function body. Spreads `fileSections`, appends `configText` and `flagText` when non-null and non-empty. Matches the plan exactly.

**D. schema.ts** — replaced the inverted-defaults stub: fixed `systemPromptSchema` to use `systemPromptSourcesSchema` for `sources` and set all defaults to `true`. Removed the stub comments.

**E. main.ts** — added `system: { type: 'string' }` to `parseArgs`, added `systemFlag`/`decodedSystem` variables next to `decodedPrompt`, added `--system` to `help.ts` usage block.

**F. main.ts** — replaced `NodeConfigWatcher` import/instantiation with `NodeDirectoryWatcher`, removed `systemPrompts` import, added `systemPromptLoader`, `resolvedSystemPrompts`, `systemPromptSessionId`, `resolveSystemPrompts` after `configLoader.start()`, wired `resolvedSystemPrompts` into `mapConfig`, called `await resolveSystemPrompts()` before first `mapConfig()`, added session-id gate in `runTurn`. Deleted `systemPrompts.ts`.

**G. RequestBuilder.ts** — replaced join-based construction with `[AGENT_SDK_PREFIX, ...(options.systemPrompts ?? [])]`, moved `cache_control` to last block only via index check in the `.map()`.
