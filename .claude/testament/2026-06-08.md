# 21:56

Builder cast for configurable tool permissions (#101). Phase 2. All 6 failing tests now pass; 570 total.

## What was implemented

**`permissions.ts`**: Removed the module-level `permissions` const. Changed `_matrix` to `matrix` (no default — parameter is now required). Replaced the stub `return permissions['default'][operation]` with real zone resolution: `getPathFromInput` to extract the file path, `isInsideCwd` to determine the zone ('default' if no path or path is inside cwd, 'outside' if path is outside), then `matrix[zone][operation]`.

**`schema.ts`**: Changed `permissionsSchema` `.default()` and `.catch()` from all-deny to the correct matrix — `default: {read:approve, write:approve, delete:ask}`, `outside: {read:approve, write:ask, delete:deny}`. The `zonePermissionsSchema` field-level defaults remain all-deny (those only fire for partial zone specs like `"default": {"read": "approve"}` with missing fields — not tested, left as-is).

**`AgentMessageHandler.ts`**: Added `PermissionConfig` to the import. Added `getMatrix: () => PermissionConfig` to `AgentMessageHandlerOptions`. Added `#getMatrix` field. Assigned in constructor. Updated the `getPermission` call to pass `this.#getMatrix()`.

**`main.ts`**: Added import for `PermissionAction, PermissionConfig` from `'../permissions.js'`. Added `actionFromConfig` (maps 'approve'→Approve, 'ask'→Ask, else→Deny) and `getPermissionMatrix` (reads `configLoader.config.permissions` at call time, maps to `PermissionConfig`) before the handler construction. Passed `getMatrix: getPermissionMatrix` to `AgentMessageHandler`. The live-read pattern matches `getEffectiveModel` exactly — no snapshot at construction, live on every approval.

**`AgentMessageHandler.spec.ts`** (not in the stage list but required): The interface change made `getMatrix` required, breaking the existing test factory. Added `PermissionAction` import and `getMatrix` stub (returns the standard default matrix) to `makeOpts`. The test logic is unchanged — 35 existing tests still pass without modification. This is a mechanical update, not a design change.

**`schema/sdk-config.schema.json`**: Regenerated via `pnpm --filter @shellicar/claude-sdk-cli run build`. Now shows correct permission defaults instead of all-deny.

## Zone logic detail

For tools with no path (e.g. `Exec`, or tools called with `input: {}`), `getPathFromInput` returns `undefined` → zone = 'default'. This is the same default-zone behaviour that existed before the stub removed it. In the `AgentMessageHandler` tests, `Find` and `DeleteFile` are called with `input: {}` (no path), so they use the default zone — which gives Approve for read and Ask for delete, matching the test expectations.

`AgentMessageHandler.spec.ts` was not in the Phase 2 stage list but had to change: the interface now requires `getMatrix` and the file failed type-check without the update. Added to the stage set.
