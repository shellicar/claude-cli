# 23:36

Courier cast for section-timestamps feature. Two phases already committed; this cast ships.

## What was built

Section dividers in the TUI now carry timing. Each divider shows:
- **Live:** `── query ── 15:29:03 ─────────────────`
- **Exited:** `── query ── 15:29:03 → 15:29:18 (15s) ──`

Duration format: seconds always shown, higher-order zero units omitted (`5s`, `2m 5s`, `1h 2m 5s`).

Two additional fixes shipped as part of iteration 2:

**Prompt timing** — the prompt divider was always reading `0s` because both enter and exit timestamps fired at submit. Fix: `ConversationState` gains `markPromptStart()`, called from `AppLayout.waitForInput()` at idle entry. `transitionBlock('prompt')` consumes the stored instant as `createdAt` and clears it.

**Error suppression** — an error fired during the query phase wrongly opened a `response` section before any response content. Fix: remove the `transitionBlock('response')` call from both error paths (`AgentMessageHandler.ts` case `'error'` and `runAgent.ts` catch block). Errors now append to whatever section is already active.

## Traps and decisions

**`@js-joda/core` and timezone.** `ZoneId.systemDefault()` works without `@js-joda/timezone`. The timezone extension is only needed for named IANA zones. Time display uses `instant.atZone(ZoneId.systemDefault()).toLocalTime().format(DateTimeFormatter.ofPattern('HH:mm:ss'))` — one library end to end.

**FakeClock needs millis() and equals().** `@js-joda/core` v5's abstract `Clock` requires both. The plan omitted them. Added `override millis() { return this.#current.toEpochMilli(); }` and `override equals(obj: unknown) { return this === obj; }`. Trivially correct for a fake.

**Double-divider problem.** Opening the prompt block in `waitForInput()` would produce two prompt dividers: one from the real block (rendered by `renderConversation.ts`) and one from `AppLayout.render()`'s separate `buildDivider('prompt', cols)` call for the editor input area. The stored-instant pattern avoids this — only `transitionBlock('prompt')` at submit opens the real block.

**Continuation suppression + timestamps.** Consecutive same-type blocks suppress headers after the first. The last block in a same-type run has `exitedAt` but its header is hidden. Accepted without special-casing.

**Error fix churn.** The R2 requirement went through five framings across casts before landing on the two-line deletion. The saga: invented an `error` BlockType (wrong); removed the transition (right); added a no-section fallback (over-scope, never runs); added `appendToTail` (over-scope); cut back to the two-line deletion. The minimal change was always the answer. Build for the stated bug, not the imagined edge.

**type-check requires workspace build first.** `pnpm -F claude-sdk-cli type-check` fails with module resolution errors unless `@shellicar/claude-core`, `@shellicar/claude-sdk`, and `@shellicar/claude-sdk-tools` are built first. Pre-existing, not introduced by this change.

**`createdAt?` optionality is intentional.** Blocks added via `addBlocks` (history replay, startup banner) have no creation instant. The `?` is correct; a JSDoc comment on the field says so.
