## The schema stub trap

`z.any()` was my first attempt for the schema stub. In Zod v4, `z.any()` without `.optional()` or `.default()` in a `z.object()` causes parsing to fail entirely when the field is absent — breaking all tests that call `parse({})`, not just the `systemPrompt` tests. Switched to a proper structured schema with wrong boolean defaults (`false` instead of `true`). This keeps the structure correct (so `initConfig.spec.ts` passes) while making the defaults/fallback tests fail.

## The unused `systemPromptSourcesSchema`

The schema file declares `systemPromptSourcesSchema` (correct all-true defaults) but the stub's `systemPromptSchema` inlines its own sources schema with false defaults. This is intentional: `systemPromptSourcesSchema` is declared with the correct shape for the Builder to reference, and TypeScript won't error on the unused declaration (no `noUnusedLocals`). The Builder will wire it in.
