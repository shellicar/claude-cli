## Mechanics that matter

- **Inter-package deps are all `workspace:^`.** So no pinned version ranges to hand-edit in
  dependents — pnpm resolves them to concrete versions at publish. Lockfile did NOT change.
- **`pnpm version` guards on a clean tree even with `--no-git-tag-version`.** The first bump
  dirties the tree, so the remaining five fail with `ERR_PNPM_UNCLEAN_WORKING_TREE`. Add
  `--no-git-checks` to the bump for all-but-the-first (or all of them).
- **Changelog regen: `pnpm --filter scripts run changelog`** regenerates all packages in one shot.
  Pre-release ⇒ no release markers, everything stays under `[Unreleased]`. The CHANGELOGs were
  already in sync from the doc PR, so regen produced **zero diff** — only the six `package.json`
  changed. Validate with `cd scripts && pnpm exec tsx src/validate-changes.ts` → "no errors found".
- **Verify suite all green:** `pnpm build`, `type-check`, `test`, `run ci` (biome, 487 files, clean).
