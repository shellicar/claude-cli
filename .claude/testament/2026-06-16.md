## What I built

One file changed: `packages/claude-sdk-tools/src/AppendFile/AppendFile.ts`. The stub `throw new Error('not implemented')` is replaced by the three-line handler from the brief, verbatim:

```ts
handler: async (input) => {
  const filePath = expandPath(input.path, fs);
  await fs.appendFile(filePath, input.content);
  return { textContent: { error: false as const, path: filePath } };
},
```

Added the `expandPath` import the Scaffolder deliberately left out (an unused import would have tripped biome on the scaffold). It sits in the same alphabetical slot `CreateFile.ts` uses: `@shellicar/claude-core/fs/expandPath` before `.../fs/interfaces`, so import ordering needs no further touch.

No try/catch, no existence check, as the brief specifies. `fs.appendFile` creates the file and any missing parent dirs when absent, so "file missing" is a normal append, not an error. Thrown OS errors are surfaced by the SDK tool runner, so a hand-written message would only restate them.
