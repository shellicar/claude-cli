# 01:32 AppendFile tool (Scaffolder, Builder, Courier)

Consolidated mission entry. `AppendFile` is the additive sibling of `CreateFile` in `@shellicar/claude-sdk-tools`: same `{ path, content }` shape, but it asserts only a suffix. It appends to an existing file and creates the file (plus any missing parent dirs) when absent. No `overwrite` flag, no existence-related errors, and no PreviewEdit/EditFile pair, because appending touches nothing that already exists.

## What was built and how it works

- `src/AppendFile/schema.ts`: `AppendFileInputSchema` (`path` plus `content`). `content` is required with no default, unlike `CreateFile` which defaults it to `''`. An empty string is still legal. There is no `overwrite` field. `AppendFileOutputSchema` is the same success/error discriminated union as `CreateFile`.
- `src/AppendFile/AppendFile.ts`: `createAppendFile(fs)` factory returning `defineTool`, `operation: 'write'`. The whole handler is three lines: `expandPath(input.path, fs)`, then `await fs.appendFile(filePath, input.content)`, then return `{ textContent: { error: false as const, path: filePath } }`. No try/catch and no existence check, by design. `fs.appendFile` creates the file and parent dirs when absent, so a missing file is a normal append rather than an error, and the SDK tool runner surfaces any thrown OS error (EISDIR, EACCES) to the model. A hand-written message would only restate it.
- `src/entry/AppendFile.ts`: `export const AppendFile = createAppendFile(nodeFs);`.

## Registration took three points, not the one the brief implied

`CreateFile` reaches the tool set through three places, and `AppendFile` mirrors all three:

1. tsup entry, automatic. `tsup.config.ts` globs `src/entry/*.ts`, so `entry/AppendFile.ts` builds with no config change.
2. `packages/claude-sdk-tools/package.json` `exports`: needs an `./AppendFile` block mirroring `./CreateFile`, or the subpath import does not resolve.
3. `apps/claude-sdk-cli/src/createAppTools.ts`: import `AppendFile` and place it in the `otherTools` array next to `CreateFile`.

## Traps for the next operator

- The worktree is not bootstrapped. A fresh worktree has root `node_modules` but no `@shellicar/*` workspace links and no built `dist/` for `claude-core` or `claude-sdk`. Until you run `pnpm install --frozen-lockfile && pnpm build`, both `type-check` and `test` fail on every file with `Cannot find module '@shellicar/...'` plus spurious TS7006. Frozen is correct: only the `package.json` `exports` map changed, no deps.
- `MemoryFileSystem.appendFile` concatenates onto an empty buffer when the key is absent, and `NodeFileSystem.appendFile` does `mkdir(dirname, { recursive: true })` then append. So the create-when-missing and create-parent-dirs contracts hold for both the fake and the real fs. The six-test contract lives in `test/AppendFile.spec.ts`.
- biome does not flag the unused `fs` factory param on the scaffold stub, so the factory stays `createAppendFile(fs)` with no `_fs` workaround.
