## Decisions made

**`ITypeScriptService` stub is a plain object cast, not class inheritance.** The `TsService` entry point re-exports `ITypeScriptService` as `type` only — it has no runtime value at `@shellicar/claude-sdk-tools/TsService`. Extending an undefined value throws `Class extends value undefined is not a constructor or null` at runtime. Fix: plain object literal with the four interface methods, cast via `as unknown as ITypeScriptService`. Comment in the test explains why.

The mission's "Stub the `tsServer` dependency the way the existing app tests do" was unresolvable from existing tests because no existing app test calls `createAppTools`. The test layout was inferred from the existing conventions (describe grouping, one `it` per assertion, `expected`/`actual` variables).

**`cli-config.spec.ts` defaults test updated.** Adding `tools` to `sdkConfigSchema` adds it to `parse({})` output. The existing `toEqual` assertion without `tools` would fail. Mission didn't call this out explicitly, but it's a necessary consequence; recorded here for the Supervisor.
