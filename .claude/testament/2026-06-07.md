# 08:39

Courier. Clock injection shipped.

## What was built

The SDK now injects a fresh `<system-reminder>` block with the current date/time into every API request, via `TurnRunner`. Files changed:

- `packages/claude-sdk/src/private/clockStamp.ts` — new: js-joda formatter returning `EEEE, d MMMM yyyy 'at' HH:mm:ss VV (xxx)` with an injectable `Clock` for deterministic tests
- `packages/claude-sdk/src/private/RequestBuilder.ts` — `systemReminder?: string` generalised to `systemReminders?: string[]`, looped injection of one `<system-reminder>` block per entry; `BetaContentBlockParam` added to import (correct type for the content array — `BetaTextBlockParam[]` would be wrong on tool-loop turns where the array holds mixed types)
- `packages/claude-sdk/src/private/TurnRunner.ts` — assembles the `systemReminders` list per turn: one-shot git delta first (from `turnInput.systemReminder`), clock stamp last; injectable `Clock` as 4th constructor param defaulting to `Clock.systemDefaultZone()`
- Tests: `clockStamp.spec.ts` (new), `RequestBuilder.spec.ts` (updated), `QueryRunner.spec.ts` (two predicates narrowed)

## The wiring depth decision

`string[]` lives only on `RequestBuilderOptions`. `TurnInput.systemReminder` and `PerQueryInput.systemReminder` stay as single strings. The clock is SDK-internal, computed at `TurnRunner` — propagating up would have required CLI changes (out of scope) and touched the `SdkQuerySummary` channel. `TurnRunner.run()` is the assembly point.

## js-joda version trap

`pnpm add @js-joda/locale_en` without a version pin resolves to `5.1.0`, which has a peer dep on `@js-joda/core@>=6.0.0`. The SDK uses `core@5.7.0` — peer dep mismatch. Pin to `^4`: `pnpm -F @shellicar/claude-sdk add @js-joda/locale_en@^4`, which resolves to `4.15.3` (same series as banananet).

## QueryRunner test gap

The Apostle's plan enumerated `TurnRunner.spec.ts` tests but missed `QueryRunner.spec.ts`. Two tests there used bare `text.includes('<system-reminder>')` predicates — too broad once the clock stamps every request. Both were narrowed to the specific reminder text each test cares about (`'stay focused'` for the git-delta one-shot test, `'be careful'` for the cached-reminders test). The fix was authorised in the mission rather than the plan.

## Manual test not done

Unit tests confirm the request body is correctly shaped. The change has not been run against the live API. A manual turn should be sent to confirm the API accepts the extra content block and the stamp renders correctly before any future operator assumes API compatibility.
