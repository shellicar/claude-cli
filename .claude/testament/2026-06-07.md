## Deviations from the plan (decided, not invented)

1. **Dropped `consumerChannel` from `runAgent`'s signature.** The plan kept it "unless the compiler flags it unused" — it is unused (cancel moved to `CancelHandler` via injection), and `noUnusedParameters` would error. Dropping it also removed the now-unused `ConsumerMessage`/`IPublisher` imports. The `main.ts` call site drops the arg.
2. **`PrimaryView` does not import `EDITOR_PREFIX_VISUAL_WIDTH`.** The plan's PrimaryView import list included it, but PrimaryView never uses it (only `EditorHandler` does, for `moveUp/DownVisual`). Importing it would be an unused import. Omitted.

Both are noted because they were judgment calls to keep the build clean; neither changes design.

## Things worth knowing for whoever comes next

- **`main.ts` ordering matters.** `CancelHandler` closes over `consumerChannel`, which is constructed in the SDK-channel block. So the whole handler/presentation/`ViewHost`/`Flasher`/`ReadLine` construction block sits *after* `consumerChannel.subscribe(...)`, and `renderer.enter(); host.renderNow()` moved there too (was `layout.enter()` earlier in startup). `using renderer` is constructed earlier (where `AppLayout` was) because `cleanup()` references `renderer.exit()`. The startup window before `ReadLine` is constructed no longer has a ctrl+c handler — acceptable (sub-second, no input expected); the plan's wiring implies this.
- **The closure-assignment `never` trap (TS).** `let x: T | null = null; promise.then(v => { x = v; }); ... x?.prop` fails with "Property does not exist on never" because CFA narrows `x` to `null` then `?.` makes the non-null branch `never`. Reading the variable *directly* (`const actual = x`) is fine — that's why the AgentMessageHandler/ApprovalHandler patterns compiled but EditorHandler's `resolved?.text` did not. Fix: capture the field inside the callback (`resolvedText = v.text`).
- **Async tool-approval in tests.** Auto-approve/deny paths in `AgentMessageHandler.#toolApprovalRequest` run synchronously (no `await` before `appendStreaming`), so state is set by the time `handle()` returns. The manual path awaits `requestApproval()`; tests resolve it via `toolApprovalState.resolveNextApproval(...)` then `await setImmediate`. The token-delta annotation lands on the *active* tools block here (never sealed in the isolated flow) — `toolsBlockContent` helper reads active-or-sealed to be robust.
- **Turbo cache is unreliable in this worktree.** First `pnpm run type-check` reported "FULL TURBO" cache hit and masked real test-file errors. Run package-level `tsc -p tsconfig.check.json` / `vitest run` directly, or `turbo run ... --force`, to actually verify.
- **Worktree bootstrap:** no `node_modules` on arrival; `pnpm install` (lockfile already up to date, ~1s).
