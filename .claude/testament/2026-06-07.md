## Spec discrepancy ÔÇö 13/16 failing (not 16/16)

The mission says "All 16 tests fail." In practice, 13 fail and 3 pass:
- Test 4: `decide('1.0.0', '1.1.0').setLatest` expected `false` ÔÇö stub returns `false` Ôćĺ coincidentally passes
- Test 6: `decide('1.0.0-beta.1', '1.0.0-beta.7').setLatest` expected `false` Ôćĺ passes
- Test 8: `decide('1.0.0-beta.1', '1.0.0').setLatest` expected `false` Ôćĺ passes

There is no sentinel value for `setLatest` that can fail tests expecting both `true` (tests 3, 5, 7) and `false` (tests 4, 6, 8) simultaneously. The mission spec is incorrect on this point. The tests themselves are correct ÔÇö they will properly exercise the Builder's implementation.

## For the Builder

The `semver` import in the stub is unused (it compiles fine ÔÇö tsx doesn't enforce `noUnusedLocals`). The Builder fills in the body of `decide` and adds the CLI entry point to the same file.

The channel validation rules the Builder needs to enforce (from tests 11ÔÇô16):
- Exactly 2 pre-release identifiers (`<name>.<number>`)
- First identifier must be non-numeric (rejects `0`, numeric-looking strings)
- Second identifier must be numeric (rejects `foo`)
- `latest` is a reserved channel name (cannot be used as a pre-release channel)
- Invalid semver should throw
