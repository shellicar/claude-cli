## What was built

Replaced `scripts/get-npm-tag.sh` (and the `scripts/tag-latest.sh` workaround) with a TypeScript decision layer that correctly handles both channel assignment and `latest` dist-tag promotion.

Two files:

- `scripts/src/decide-dist-tags.ts` — pure function, no I/O. Exports `decide(version, currentLatest) → { channel, setLatest }`. This is what the 16 tests cover.
- `scripts/src/decide-cli.ts` — CLI wrapper. Reads `--package-dir` and `--current-latest`, reads `version` from `<package-dir>/package.json`, calls `decide`, prints `key=value` lines to stdout.

Workflow: two steps in `.github/workflows/npm-publish.yml`.
- "Decide dist-tag" (id: `dist-tag`): queries `npm view` for current latest, runs the CLI redirecting stdout to `$GITHUB_OUTPUT`, appends `pkg=$PKG`.
- "Publish": uses `${{ steps.dist-tag.outputs.channel }}`, `${{ steps.dist-tag.outputs.version }}`, `${{ steps.dist-tag.outputs.setLatest }}`.

## Decision logic

`semver.gt` handles stable-vs-prerelease naturally: a stable version is always greater than a pre-release in the semver spec, so `semver.gt('1.0.0', '1.0.0-beta.1')` is `true` and `semver.gt('1.0.0-beta.1', '1.0.0')` is `false`. No special casing needed for mixed types — one expression covers all four cases.

Validation flow for pre-release versions:
1. `prerelease.length !== 2` — catches `rc.foo.5` (3 segments), bare `beta` (1 segment), `0` (1 segment)
2. `typeof prerelease[0] !== 'string'` — catches numeric-only first identifier
3. `typeof prerelease[1] !== 'number'` — catches non-numeric second identifier (`beta.foo`)
4. `prerelease[0] === 'latest'` — reserved name guard

## Path resolution trap

`pnpm --filter scripts exec tsx src/decide-cli.ts` changes the process CWD to `scripts/`. Relative paths like `packages/claude-core` resolve to `scripts/packages/claude-core` — wrong.

`INIT_CWD` is not set by `pnpm exec` (only by `pnpm run`). `process.env.PWD` gets updated by tsx to match the new CWD and is therefore unreliable.

Fix: `resolve(dirname(fileURLToPath(import.meta.url)), '..', '..')`. The script lives at `scripts/src/decide-cli.ts`, so two levels up from `import.meta.url` is always the repo root. Deterministic, no env var dependency.
