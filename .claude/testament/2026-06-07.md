# 04:29

Phase 4 Apostle. Plan for the backoff clock-seam fix.

## The problem

`defaultSleep` hardcodes `setTimeout`. The two `defaultSleep` tests assert correctness by measuring wall-clock elapsed time (`Date.now()` deltas with `< 10` and `< 100` bounds). On a loaded CI runner, the already-aborted case took 11ms against the 10ms bound: `expected 11 to be less than 10`. The implementation is correct; the test instrument is wrong.

## The seam

Added `ScheduleTimer = (ms: number, onExpiry: () => void) => () => void` type. Added unexported `defaultTimer` (wraps `setTimeout`/`clearTimeout`, returning a cancel function). Added optional third parameter `schedule: ScheduleTimer = defaultTimer` to `defaultSleep`. Body replaces direct `setTimeout` call with `schedule(ms, resolve)` and the returned `cancel` function in the abort handler.

The cancel-function shape hides the `setTimeout` handle entirely from the seam. Tests never need to construct a `NodeJS.Timeout` or cast to satisfy `clearTimeout`'s type.

## The tests

`neverFires = (_ms, _onExpiry) => () => {}` — never calls the callback, no-op cancel.

With `neverFires`, the only path to `resolve()` is the abort-signal listener. Both tests assert resolution deterministically:
- Already-aborted: `defaultSleep` returns `Promise.resolve()` immediately; `schedule` is never called.
- Fires during sleep: enters Promise body, `neverFires` registers nothing, `controller.abort()` triggers the listener.

Both use `.then(() => 'resolved', () => 'rejected')` with `expected`/`actual` constants per the TDD skill.

## Callers

Only one caller outside tests: `TurnRunner.ts` line 57 (`this.#sleep = sleep ?? defaultSleep`). Field typed as `(ms, signal) => Promise<void>`. The updated `defaultSleep` with optional third param is assignable — no change to `TurnRunner.ts`. `TurnRunner.spec.ts` injects its own sleep stub at constructor level; unaffected.

## What the Maker needs to know

- The `ScheduleTimer` type is local to `backoff.ts`, not exported — it's an implementation detail of the seam, not public API.
