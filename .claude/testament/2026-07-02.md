## Markdown render in the TUI, shipped (mission 2026-06-29_markdown-render)

Assistant `response` blocks now render as styled ANSI markdown, opt-out via config.
The code and the diff show how it is wired. What follows is what they do not say.

### Decisions and why

- **Markdown is response-only, not a global swap.** The obvious move is to route all
  block content through the markdown renderer. Do not. `CODE_FENCE_RE` in `blockLayout.ts`
  serves tool JSON, notices, and thinking too; sending those through `marked` regresses
  their fence highlighting. The gate is `markdown.enabled && block.type === 'response'`.
- **Streaming needed no new seam.** Primary repaints from state every frame, so a sealed
  block re-renders itself once complete. Streaming got its own flag (`markdown.streaming`,
  gated under `enabled`) so it can be turned off without losing sealed rendering.
- **The count-preservation invariant is load-bearing.** `markdownLayout` mirrors
  `blockContentLines`: the code `decorate` returns one line per source line. Break that and
  any plain-measurement path disagrees with the highlighted render, and heights drift. If
  you extend this, keep decorate count-preserving.
- **History and thinking markdown were deferred to a follow-up** (SC call on the plan's open
  question). The layout lives in `model/` precisely so that follow-up can reach it from the
  History measurement path without moving code.

### What the SC corrected in live testing

The first build passed its tests and still had two visible code-box defects, both in `box()`:

- The border width was `inner - 5`, which only aligns for a 2-char label. `ts` and `md`
  looked right because the correct formula (`innerW - 1 - labelWidth`) coincides at width 2;
  `plaintext` overshot. Tests that only used 2-char labels could not catch this. The lesson:
  the box unit test now pins exact bytes by hand rather than comparing `box` against itself,
  so it catches a formula regression instead of being a tautology.
- Long code lines ran off the right edge. `box` now caps to `termWidth - 4` and wraps.

### Traps that will waste your time

- **`marked`'s `space` token must emit one blank line, not `break`.** The plan said `break`.
  Drop the blank and the heading/blank/code-box layout collapses.
- **Wrap code bodies with `wrapLine` (claude-core reflow), not the spec's `hardWrap`.** Real
  code carries SGR escapes; `hardWrap` byte-slices and will cut an escape in half. They agree
  on plain ASCII, which is why the spec's plaintext case still matches byte-for-byte.
- **The palette owns its own codes; do not reach for claude-core's.** The spec's `dim` is
  `\x1b[90m` (bright-black fg), not claude-core `DIM` `\x1b[2m`. The spec output is the contract.
- **Keep vite on `^7`.** `pnpm add marked` (on `^18`) left it there; a careless update to v8
  breaks the decorator toolchain. See project CLAUDE.md.
# 03:47 — Interruption resilience, shipped (Courier, Phase 7)

This PR (`feature/interruption-resilience`) makes the CLI survive a mid-turn network
drop — the `[terminated]` undici error when the laptop sleeps with a request in flight.
Before this, the process lived but the turn was lost, and because the conversation was
only persisted per query (not per turn), history could roll back to the last save. Four
pieces, built as one cohesive change (Maker commits `8856eea` + `1bb889d`, both by the SC).
This entry replaces the phase-by-phase design logs; the useful residue is below.

## What shipped, and where the design is load-bearing

**1. Wake lock (macOS).** `IWakeLock` (abstract token in `claude-sdk/src/public/interfaces.ts`)
← `PlatformWakeLock` (CLI `src/model/`). Held across the whole request inside
`TurnRunner.run` (a `try { for(;;){…} } finally { wake.release() }`), because the request
lifecycle — connect, backoff, stream — lives entirely in `TurnRunner`, and `message_start`
only fires after the connection succeeds, too late to cover connect/backoff. The design was
pinned by the SC after three rejected iterations; do not re-derive it:

- DI knows **one** mapping, `IWakeLock → PlatformWakeLock`. That impl's `acquire()` **is** the
  factory — it produces the handle.
- The platform choice is a `switch (…)` **in `acquire()`, in code, never in DI**. Adding
  linux/windows is a new `case` + a new mechanism, not a new registration.
- Each platform mechanism owns its own invocation. caffeinate's `-i -w <pid>` lives in
  `caffeinateWakeLock` and nowhere else (`-w` is the anti-leak guard: a crash without release
  can't strand the machine awake).
- `preventSleep` config is read **live inside `acquire()`**, so toggling it mid-session takes
  effect. This is the crux: core-di-lite registrations are all singletons, so any per-use
  decision must sit in a method called per-use, never in the registration factory (that runs
  once and freezes config). The two earlier designs both failed by resolving statically —
  a startup factory, then a mechanism class with caffeinate's flags hardcoded.
- `PlatformWakeLock` reads the platform via `IFileSystem.platform()`; the enabled+darwin
  spawn test is macOS-host-gated (`process.platform` has no provider abstraction anywhere in
  the repo — left that way to avoid unrequested infra).
- `IWakeLockSpawner`/`NodeWakeLockSpawner` is a new killable-spawn seam (the existing
  `IProcessLauncher` is fire-and-forget, no kill handle). It swallows a spawn ENOENT so a box
  without caffeinate never errors — that swallow replaces a startup PATH probe.

**2. Retry the mid-stream drop (SDK).** In `transport.ts`, a raw undici `terminated` (it
escapes `parseSse` uncaught) is wrapped as `StreamInterruptedError`, which extends
`TransportError` **not** `ConnectionError` — so it bypasses the generic exponential
`isRetryable` path and takes its own bounded short-fixed retry branch in `TurnRunner`
(~2 attempts, ~30s fixed), logged at `warn`. Note `ApiStreamError extends TransportError`
too, so transient SSE-frame errors still pass through the same `instanceof TransportError`
gate unwrapped.

**3. Seal the partial reply on reconnect (not discard).** `StreamInterruptNotice.reconnecting()`
calls `ConversationState.completeActive()` (the existing seal) then `spliceNotice()`. Sealing
keeps the partial output on screen and fixes the display garble (the retry can't append onto
a sealed block). Hiding it was an earlier wrong turn the SC corrected. Wired only to the
stream-interrupt branch; wiring it to generic transient retries is left open (would show a
spurious "connection dropped" on a pre-stream 500).

**4. Per-turn persistence + submit-to-resume (CLI).** The conversation is saved both when a
user message is **sent** (the `query_summary` hook in `AgentMessageHandler` — it fires once
per turn before the interruptible request, and the just-sent user message, end-human or
`tool_result`, is already in the `Conversation`, so one hook covers both) and after the
assistant reply (`turn_content`). Both use the same atomic temp-then-rename write, which
needed a new `IFileSystem.rename`. An empty submit resumes: `EditorHandler` allows it when
`messages.at(-1)?.role === 'user'`, and `runAgent` sends `messages: []`, which
`QueryRunner.run` loops against the existing trailing user message — no SDK change.

## The one architectural fact to hold before touching this codebase

Everything is **core-di-lite property injection** (repo-wide since #378). Classes declare deps
as `@dependsOn(Token) private readonly x!: T` fields with **no constructor**; `container.ts`
registers `abstraction → concrete` with a bare `.to(Impl)` and auto-wires. Hand-written
`(x) => new X(...)` factories survive only where there's real composition-root work (assembling
handler chains, reading config to pick a db file). `AccountLimitListener` is the exact mirror
both new SDK seams copy (abstract class in `public/types.ts`, two container registrations,
injected into `TurnRunner`). Verify wiring against the actual `container.ts` — grounding
against a remembered DI shape was caught wrong repeatedly on this mission.

## Traps that cost real time

- **Adding a `@dependsOn` field breaks every test that builds that class through a container.**
  Eager `buildProvider()` injects on all registered instances, so a missing contract throws
  `UnregisteredServiceError` at build time — the compiler won't catch it, only the test run.
  Beyond the obvious specs, `test/ThinkingRequestWiring.spec.ts` has its own `buildTurnRunner`
  and `test/ViewHost.spec.ts` builds the editor chain. After adding a field, grep all of
  `test/` for `register(<thatClass>)` and every helper that resolves it.
- **The JSON config schema is generated.** Edit the zod `schema.ts` only; `pnpm build`
  regenerates `schema/sdk-config.schema.json`. Hand-editing drifts and is overwritten.
- **`cli-config.spec.ts` has a strict `toEqual` on the whole default config object** — a new
  schema field breaks it unless the expected object is updated (whole-object equality, easy to miss).
- **biome runs only in `pnpm run ci`, never in `type-check` or `test`.** It will fail CI on
  formatting/import-sort/member-accessibility that a green type-check and test sailed past.
  Run it before pushing; fix with `pnpm ci:fix`, never `biome check --write`.
- **The dated testament files are `.md merge=union` and shared across concurrent missions.**
  Re-staging one can make it *look* like you replaced another mission's entries; the union
  driver recombines both sides at merge. Keep your PR's legacy in your own dated file, and
  before shipping run `git diff origin/main...HEAD -- .claude/testament` to confirm you only
  touched your own. (This PR restored `2026-06-29.md`/`2026-06-30.md` to main's content for
  exactly that reason.)

## Process lesson from the mission (the expensive one)

Phase 4 was a Maker with no design phase — it improvised the wake-lock's DI (config + platform
logic in the `container.ts` factory, dynamic config snapshotted at construction, a concrete
registered to itself). The SC rejected it on quality and reverted the whole build. An Apostle
design phase was inserted (Phase 5), and even then the wake-lock took three iterations plus a
codebase move (#378 landed underneath it) before the SC pinned the structure in conversation
and relayed it as locked constraints. Investigation facts are not a design; a build with real
design choices needs the design settled and approved first.
