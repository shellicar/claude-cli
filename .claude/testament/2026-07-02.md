## Markdown render in the TUI, shipped (mission 2026-06-29_markdown-render)

Assistant `response` blocks now render as styled ANSI markdown, opt-out via config.
The code and the diff show how it is wired. What follows is what they do not say.

### Decisions and why

- **Markdown is response-only, not a global swap.** The obvious move is to route all
  block content through the markdown renderer. Do not. `CODE_FENCE_RE` in `blockLayout.ts`
  serves tool JSON, notices, and thinking too; sending those through `marked` regresses
  their fence highlighting. The gate is `markdown.enabled && block.type === 'response'`.
- **Streaming needed no new seam.** Primary repaints from state every frame, so a sealed
  block re-renders itself once complete. Streaming got its own flag (`markdown.streaming`,
  gated under `enabled`) so it can be turned off without losing sealed rendering.
- **The count-preservation invariant is load-bearing.** `markdownLayout` mirrors
  `blockContentLines`: the code `decorate` returns one line per source line. Break that and
  any plain-measurement path disagrees with the highlighted render, and heights drift. If
  you extend this, keep decorate count-preserving.
- **History and thinking markdown were deferred to a follow-up** (SC call on the plan's open
  question). The layout lives in `model/` precisely so that follow-up can reach it from the
  History measurement path without moving code.

### What the SC corrected in live testing

The first build passed its tests and still had two visible code-box defects, both in `box()`:

- The border width was `inner - 5`, which only aligns for a 2-char label. `ts` and `md`
  looked right because the correct formula (`innerW - 1 - labelWidth`) coincides at width 2;
  `plaintext` overshot. Tests that only used 2-char labels could not catch this. The lesson:
  the box unit test now pins exact bytes by hand rather than comparing `box` against itself,
  so it catches a formula regression instead of being a tautology.
- Long code lines ran off the right edge. `box` now caps to `termWidth - 4` and wraps.

### Traps that will waste your time

- **`marked`'s `space` token must emit one blank line, not `break`.** The plan said `break`.
  Drop the blank and the heading/blank/code-box layout collapses.
- **Wrap code bodies with `wrapLine` (claude-core reflow), not the spec's `hardWrap`.** Real
  code carries SGR escapes; `hardWrap` byte-slices and will cut an escape in half. They agree
  on plain ASCII, which is why the spec's plaintext case still matches byte-for-byte.
- **The palette owns its own codes; do not reach for claude-core's.** The spec's `dim` is
  `\x1b[90m` (bright-black fg), not claude-core `DIM` `\x1b[2m`. The spec output is the contract.
- **Keep vite on `^7`.** `pnpm add marked` (on `^18`) left it there; a careless update to v8
  breaks the decorator toolchain. See project CLAUDE.md.

# 04:24 — Interruption resilience (Courier, Phase 7)

Shipped in PR #386: the CLI survives a mid-turn network drop (the undici `terminated` on
sleep/wake). Four pieces — macOS wake-lock, SDK retry of the drop, seal-on-reconnect, per-turn
persistence + submit-to-resume. The diff and PR tell that story; below is only what they don't.

## The wake-lock design is pinned — don't re-derive it

It took three rejected design iterations to land here, so if you're tempted to restructure it,
these are the constraints that were fought for:

- DI knows **one** mapping, `IWakeLock → PlatformWakeLock`. That class's `acquire()` **is** the
  factory — it returns the handle.
- The platform choice is a `switch` **inside `acquire()`, in code — never in DI.** Adding
  linux/windows is a new `case` + mechanism, not a new registration.
- `preventSleep` config is read **live inside `acquire()`**. This is the crux: core-di-lite
  registrations are all singletons, so any decision that must track dynamic config has to sit in
  a per-call method, never in the registration factory (which runs once and freezes it). Both
  earlier designs died by resolving statically.
- Each platform mechanism owns its own invocation — caffeinate's `-i -w <pid>` lives only in
  `caffeinateWakeLock`.

## The one fact to hold before touching this codebase

Everything is **core-di-lite property injection**: `@dependsOn(Token) private readonly x!: T`
fields, **no constructor**; `container.ts` registers `abstraction → concrete` with a bare
`.to(Impl)` and auto-wires. `AccountLimitListener` is the mirror the SDK seams copy (abstract
class in `public/types.ts`, two registrations, injected into `TurnRunner`). Verify wiring against
the actual `container.ts` — grounding against a remembered DI shape was caught wrong repeatedly.

## Traps that cost real time

- **Adding a `@dependsOn` field breaks every test that builds that class through a container.**
  Eager `buildProvider()` injects on all registered instances → `UnregisteredServiceError` at
  build time, which the compiler won't catch, only the test run. After adding a field, grep all
  of `test/` for `register(<thatClass>)` and every helper that resolves it (there are stray
  builders in `ThinkingRequestWiring.spec` and `ViewHost.spec`, not just the obvious specs).
- **biome runs only in `pnpm run ci`** — never in `type-check` or `test`. It will fail CI on
  import-sort / `import type` / formatting that a green type-check and test sailed past. Run it
  before pushing; fix with `pnpm ci:fix`, never `biome check --write`.
- **The JSON config schema is generated** by `pnpm build`. Edit the zod `schema.ts` only.
- **`cli-config.spec.ts` has a strict `toEqual` on the whole default config** — a new schema
  field breaks it until the expected object is updated.
- **Testament files are `.md merge=union`, shared across concurrent missions.** Re-staging one can
  look like you replaced another mission's entries; the union driver recombines both sides at
  merge. Keep your legacy in your own dated file, and check
  `git diff origin/main...HEAD -- .claude/testament` shows only yours before shipping.

## Where the `terminated` retry lives

`transport.ts` wraps the raw undici `terminated` (it escapes `parseSse` uncaught) as
`StreamInterruptedError`, which extends `TransportError` **not** `ConnectionError` — so it
bypasses the generic exponential `isRetryable` path and takes its own bounded fixed-delay branch
in `TurnRunner`. If you touch retry classification, that base-class choice is load-bearing.
