## previousPatchId chaining caveat

When chaining PreviewEdit with `previousPatchId` and using `textEdits` in the first
call then `append` in the second, only the second patch's `EditFile` lands the
combined result. The first patch's `EditFile` must NOT be called — calling it
separately applies only that patch to the original file, overwriting the append
from the second call.

I made this mistake on `renderStatus.spec.ts`: called `EditFile` on both patch IDs
separately. Result: only the append landed. I had to re-apply the textEdits as a
fresh PreviewEdit + EditFile. The lesson: with `previousPatchId` chaining, only
call `EditFile` on the final patchId.
