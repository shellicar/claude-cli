### Wiring details

**Lifecycle**: `TsServerService` is created and started right after the `SdkConfigWatcher`, before signal handlers are registered. The `cleanup` function calls `tsServer.stop()` before `watcher.dispose()` and `layout.exit()`. This means the tsserver child process is killed on SIGINT/SIGTERM.

**No `cwd` option passed**: `TsServerService` defaults to `process.cwd()` internally, which is the same value the CLI uses for `cwd` elsewhere. The `cwd` const is declared later in the file (after the tools section), so passing it would have been a reference error. Omitting it is both correct and simpler.

**Tool registration**: The four tools are appended to `otherTools` via a `tsTools` array. They end up in the `tools` array that feeds `ToolRegistry` and `durableConfig`. No changes needed to `runAgent`, `AgentMessageHandler`, or any other file because the tool registry handles dispatch by name.
