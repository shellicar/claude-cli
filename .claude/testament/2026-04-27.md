## 1.0.0-beta.5 release

### What shipped

Four packages bumped to `1.0.0-beta.5`: `apps/claude-sdk-cli`, `packages/claude-sdk-tools`, `packages/claude-sdk`, `packages/claude-core`. CLAUDE.md gained the release process documentation (lockstep versioning, pre-release process, build versioning). `npm-publish.yml` gained a `workflow_dispatch` trigger with dry-run support.

### GitVersion works via gh release create

`gh release create` creates the tag before the workflow starts. GitVersion sees the tag and resolves the correct version. Earlier releases appeared broken because the tag was created after the workflow ran, which meant GitVersion never saw it. That was a sequencing error. Creating releases via `gh release create` has the right sequencing by design.

### workflow_dispatch dry-run

`npm-publish.yml` accepts a `workflow_dispatch` trigger with a `tag` input. When triggered this way, the publish step runs with `--dry-run`. Use this to verify the workflow without actually publishing.

### Worktree trap

`main` may be checked out in another worktree, causing `git switch main` to fail. If it does, stop and ask the SC to resolve it. Do not attempt to fix it yourself.
