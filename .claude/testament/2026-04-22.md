# Hook improvements (#303) — the abridged story

Two deliverables:

1. Hook `command` accepts `~`, `$HOME`, and relative paths. Relative paths resolve against the **config file's directory**, not the CLI's cwd.
2. Hook approval-request JSON delivered via **stdin**, not `$1`. Shell expansion can no longer reach `$` or backticks in the payload.

Path resolution lives in the config system. Any future consumer that needs path fields uses the same mechanism — the hook is the first caller, not a special case.

## Shape that landed

**`ConfigLoaderOptions`** gained two fields:

- `fs: IFileSystem` — required. Supplies `homedir()` and `getEnvVar()` for `expandPath`.
- `pathFields?: readonly (readonly string[])[]` — optional list of nested segment paths. `main.ts` declares `[['hooks', 'approvalNotify', 'command']]`.

`ConfigLoader.#readAll()` walks each declared path in each source's parsed JSON and rewrites string leaves in place to `path.resolve(dirname(sourcePath), expandPath(value, fs))`. Reload goes through the same method, so resolution re-runs for free. The stored `ConfigSource.raw` sees the post-resolution string; existing source-tracking tests don't exercise path fields, so this is invisible to them. A future source-reader that wants the author-on-disk string needs a different affordance.

**`expandPath`** lives in `packages/claude-core/src/fs/expandPath.ts` (moved from `claude-sdk-tools`). Signature `(value, fs)`. Env var access is injectable via `IFileSystem.getEnvVar` — no `process.env` reach-through.

**`IProcessLauncher`** reshaped to `launch(command, options: { args?; stdin? })`. No dual form, no back-compat. `NodeProcessLauncher` pipes stdin with `stdio: ['pipe', 'ignore', 'ignore']`, attaches an `error` handler to `child.stdin` (EPIPE if the child exits early), then `end(stdin)`. Keeps `detached` + `unref()` for fire-and-forget.

**`resolve(dirname, expandPath(...))`** in that order: `expandPath` first so `~` and env vars become part of the candidate; `path.resolve(sourceDir, candidate)` then handles absolute pass-through for free. All four shapes (relative, absolute, `~`, `$VAR`) flow through one line.

## Traps and SC rulings

**Phase 1 shipped `pathFields: readonly string[]` with "top-level" framing.** The motivating case is nested (`hooks.approvalNotify.command`). If you take the scaffold literally, `main.ts` can't wire the bug fix end-to-end. Phase 2 stopped and asked; SC: the scaffold was wrong. Flipped to `readonly (readonly string[])[]` — cleaner than dotted strings (no parsing step, no dot-in-key collision, structure matches the walk). If a scaffold's tests only cover a degenerate case of the feature, stop and ask. Do not extend it silently; do not ship green-but-useless.

**`fs: IFileSystem` is required, not optional.** Phase 1 initially tried to make it optional to avoid touching 13 call sites. SC caught it: if it's optional, the feature doesn't work. When you say you'll add something as required, do that — don't softly downgrade to preserve compatibility unless the SC said so.

**Pre-existing `ci:fix` failures are not automatic scope.** `pnpm run ci` surfaced 14 import-sort failures in files the SC's prep commits had left dirty (`claude-sdk-tools/**` plus three in `claude-sdk-cli`). Stop, report the list, let the SC decide. Do not run repo-wide `ci:fix` on your own authority — project guidance explicitly flags that it touches files outside scope. The SC ran it themselves and those fixes rode into this PR.

**`resolvePathField(obj, segments, transform)` has three early-return branches**, all intentional:

- Empty `segments` → no-op (explicit guard, not silent zero-loop success).
- Non-object intermediate (`null`, primitive, array) → return. A declared path field absent from this source is not an error; another source can fill it.
- Leaf not a string → leave alone. Type mismatches surface at `schema.parse`, not here.

## No mocks

All I/O through abstractions with test fakes. `MemoryFileSystem` in `packages/claude-core/test/` is minimal: `homedir()`, `getEnvVar()`, `setEnvVar()`; every other method throws. `claude-sdk-tools` has its own richer `MemoryFileSystem` in `test/`. If you reach for `process.env`, `vi.mock`, or `vi.spyOn`, stop. Find the abstraction or report there isn't one.
