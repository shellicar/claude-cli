## What was implemented

Five source files, two test files. Purely additive — nothing existing was removed or changed in behaviour.

**`packages/claude-sdk/src/private/types.ts`**: Three new events added to `MessageStreamEvents`:
`tool_use_start`, `server_tool_use_start`, `tool_use_input_delta`.

**`packages/claude-sdk/src/private/StreamProcessor.ts`**: `process()` gains a `Set<number>` of
active tool-input block indices. `content_block_start` for `tool_use` and `server_tool_use` adds
the index and emits the start event. `content_block_stop` removes the index. A new `inputJson`
subscriber emits `tool_use_input_delta` when the set is non-empty. `mcp_tool_use` deltas are
silently dropped by never entering the set — matches the existing no-case posture in `contentBlock`.

**`packages/claude-sdk/src/public/types.ts`**: Three new message types (`SdkToolUseStart`,
`SdkServerToolUseStart`, `SdkToolUseInputDelta`) and updated `SdkMessage` union.

**`apps/claude-sdk-cli/src/entry/main.ts`**: Three new processor → sdkChannel forwards, placed
alongside the existing `server_tool_use` and `server_tool_result` forwards.

**`apps/claude-sdk-cli/src/controller/AgentMessageHandler.ts`**: Three new switch arms inserted
after `server_tool_result`. `tool_use_start` and `server_tool_use_start` both transition to tools
block, snapshot `usageBeforeTools`, and stream the name (with `🌐 ` prefix for server). 
`tool_use_input_delta` streams the raw JSON — no transition, no snapshot.

because the start event placed it. The set is the minimum needed to gate `inputJson` forwarding.
