# 20:58 (Courier)

Thinking-toggle feature shipped. What follows is what future operators need that the code doesn't say.

## What was built

- `thinking` config object in `sdk-config.json`: `enabled` (boolean, default `true`) and `effort` (enum `low|medium|high|xhigh|max`, default `max`).
- Session-local overrides for both: thinking cycles none/on/off, effort cycles none/max/xhigh/high/medium/low. Neither writes to disk.
- `m` key in command mode opens a model-settings sub-mode. Inside: `t` cycles thinking, `e` cycles effort. ESC pops one level; Ctrl+/ exits from any depth.
- Status line: `*thinking` / `*no thinking` and `*effort:max` (only when override is set).
- SDK: `ThinkingEffort` type on `DurableConfig.thinkingEffort`, wired to `body.output_config.effort` in `RequestBuilder`.

## Traps and decisions

**`xhigh` is not in the Anthropic SDK types.** `BetaOutputConfig.effort` union is `'low' | 'medium' | 'high' | 'max' | null` — no `xhigh`, even in 0.98.0. `xhigh` is a real API value (introduced with extended thinking), the API is ahead of the types. `RequestBuilder.ts` casts at the boundary with a documented comment. The `ThinkingEffort` type in this repo correctly includes `xhigh`; the cast is only at the SDK call site. When a future SDK update adds `xhigh` to the union, the cast can be removed.

**`effort` goes on `body.output_config`, not `body.thinking`.** Effort applies to all token spend, not just thinking tokens. `BetaOutputConfig.effort` is the right field; the `thinking` block is separate.

**Callback wiring in `main.ts` must come after the closures.** `layout.setThinkingCycle(cycleThinkingOverride)` and `layout.setEffortCycle(cycleEffortOverride)` must land after the `const cycleThinkingOverride` and `const cycleEffortOverride` declarations, not immediately after `new AppLayout()`. Placing them too early is a temporal dead zone error at runtime. The calls land just before `configLoader.onChange`.

**`ThinkingEffort` must be re-exported from `packages/claude-sdk/src/index.ts`.** It is defined in `types.ts` and imported by both `main.ts` and `StatusState.ts` via `@shellicar/claude-sdk`. The Maker plan's Section 8 did not list `index.ts`; the Maker added the export as a necessary gap fill.

**Sub-mode design: no existing root bindings move.** Earlier iterations assigned `t` to thinking at the root, displacing the clipboard-text paste binding. After two failed iterations the design settled on `m` at root for a model-settings sub-mode, with `t` and `e` inside it. The rule: no existing root-level command-mode bindings may be displaced. Hold this rule for any future key-binding work.

**Root hint now shows `m` and `n`.** Before this work, `n new` was in `#handleCommandKey` but absent from the hint. The block before final Maker approval caught this. Both `m model` and `n new` are now in both root hint variants, locked by tests in `renderCommandMode.spec.ts`.

**Effort override displays when thinking is off.** The SC reviewed and accepted this during manual testing. Do not fix it without a fresh decision.

**`configLoader.onChange` needs no edit.** Resolvers re-read `overrides.*` at call time, so config reload cannot clobber the override. The existing `Object.assign(durableConfig, mapConfig())` pattern in `main.ts` is called before every turn.

## Key file locations

- Override slot: `main.ts` `overrides` object (after `const layout = new AppLayout(...)`)
- Resolvers: `getEffectiveThinkingEnabled`, `getEffectiveEffort`, `getEffectiveModel` — all closures in `main.ts`
- Cycle functions: `cycleThinkingOverride`, `cycleEffortOverride` — closures in `main.ts`
- Sub-mode state: `apps/claude-sdk-cli/src/model/CommandModeState.ts` (`#context: 'root' | 'model'`)
- Command dispatch: `AppLayout.#handleCommandKey` (root) and `AppLayout.#handleModelKey` (model sub-mode)
- Status slots: `apps/claude-sdk-cli/src/view/renderStatus.ts` `renderModel` function
- Hint strings: `apps/claude-sdk-cli/src/view/renderCommandMode.ts` `buildCommandRow` function
- API wiring: `packages/claude-sdk/src/private/RequestBuilder.ts` (`body.output_config.effort`, cast documented)
- Config schema: `apps/claude-sdk-cli/src/cli-config/schema.ts` (`thinkingSchema`); auto-generated JSON Schema at `schema/sdk-config.schema.json`
