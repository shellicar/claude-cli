## What the block identified

**Pattern 1 — inline expect.** The four `statusBar` tests in `cli-config.spec.ts`
used `expect(value).toBe(literal)` without `expected`/`actual` variables. Resolved
automatically by the Pattern 2 fix (the tests were dropped entirely).

**Pattern 2 — tautological tests that pass at scaffold time.** Seven tests across
two files passed against stubs because they asserted initial values or schema
declarations rather than implemented behaviour.

## Decisions made in the plan

**The four `statusBar` cli-config tests: dropped.**

The choice was binary: stub the schema for the Builder (so tests fail at scaffold)
or drop the tests (schema defaults are declarations). I chose drop. The reason:
Zod `.default()` and `.catch()` values are the specification. A test that asserts
`parse({}).statusBar.showConversationId === true` simply restates the schema line.
There is no independent implementation path for the test to probe. If the default
changes, the schema line changes — no failing test would alert to the regression
because the test would be updated in lockstep with the schema. These tests had no
signal to add.

**The three `StatusState.spec.ts` tests: dropped.**

- `isModelOverridden starts false` and `showConversationId starts false`: field
  initialisers are declarations, not behaviours. The field declaration IS the
  assertion. Dropped.
- `setModel(name) defaults isModelOverridden to false`: passes vacuously at
  scaffold time because `setModel` never stores the flag and the field never
  changes from its initial `false`. The test only becomes meaningful after the
  positive case (`setModel(name, true)`) is implemented, at which point it adds
  no incremental signal — the positive case already pins the setter's effect.
  Dropped.

## What stays in the plan

All test code in the plan uses `expected`/`actual` variables. The two remaining
`StatusState` tests (`setModel(name, true) sets isModelOverridden to true` and
`setShowConversationId(true) flips it on`) each fail because the stubs do not
implement the setter behaviour.
