Iteration 3 of Phase 1. Design fully replaced: `m` sub-mode pattern.

## What changed from iteration 2

Iteration 2 had: root `t` → thinking cycle; `t`→`p` reassignment; `p` preview displaced; effort config-only.

Iteration 3 has: `m` at root enters model sub-mode; `t` and `e` inside the sub-mode; no existing bindings move; effort is now also session-overridable.

## Sub-mode pattern

`CommandModeState` gains `#context: 'root' | 'model'`. The legacy `apps/claude-cli/src/CommandMode.ts` already uses `_context: 'root' | 'session'`; same shape applied to sdk-cli. Key points:

- `m` at root → `enterModelSubMode()` → context becomes `model`
- `t` inside model → `#onThinkingCycle?.()` (callback from main.ts)
- `e` inside model → `#onEffortCycle?.()` (callback from main.ts)
- ESC: if context is `model` → `exitModelSubMode()` (pop to root); if context is `root` → `exitCommandMode()` (exit entirely)
- Ctrl+/ → `toggleCommandMode()` (modified to reset context when turning off)

`#handleCommandKey` in AppLayout branches on context first. The model branch goes to `#handleModelKey`.

## Effort is now session-overridable

`overrides` gains `effort: ThinkingEffort | null`. `getEffectiveEffort()` resolver. `cycleEffortOverride()` cycles null → max → xhigh → high → medium → low → null. `thinkingEffort: getEffectiveEffort()` in `mapConfig()` (not directly from config anymore).

## Two status slots

`StatusState` gets `#thinkingOverride` and `#effortOverride`. `renderModel` appends both when set. Effort format: `*effort:max` — label + value, both visible.

## Root hint unchanged

The brief says "at the root, the existing hint applies." Plan leaves root strings as-is. `m` is undiscoverable from the hint (same as `n`). Plan note #2 says if SC wants `m` visible it's a one-line edit.

## `m` is free

Verified by reading all `case` entries in `#handleCommandKey`: `t`, `f`, `i`, `d`, `p`, `n`. No `m`. No `e` either. Both are free at root; `e` only appears inside model sub-mode.

## What was implemented

Followed the plan exactly with one deviation and one gap filled.

**Deviation: callback wiring placement.** The plan places `layout.setThinkingCycle(cycleThinkingOverride)` "after layout construction". That prose was ambiguous. I initially placed the calls immediately after `new AppLayout(...)` — which is before `cycleThinkingOverride` is declared as a `const`. That would be a temporal dead zone violation at runtime. Caught during preview, corrected: the calls land after all `const` declarations (resolvers, cycle arrays, callbacks), immediately before `configLoader.onChange`. The Apostle plan's "after layout construction" maps to "after the closures that the callbacks reference are defined", not "immediately after the new call".

**Gap filled: `packages/claude-sdk/src/index.ts`.** The plan says to import `ThinkingEffort` from `@shellicar/claude-sdk` in `main.ts` and `StatusState.ts`, but Section 8 (files changed) does not list `index.ts`. Without exporting `ThinkingEffort` from `index.ts`, the import would fail. Added both the private import and the public re-export. This is a necessary consequence of adding the type to `types.ts`.

**Auto-generated schema.** `schema/sdk-config.schema.json` updated automatically when the Zod schema gained the `thinking` field. Staged it alongside the source change — it's the file that IDEs use for config validation.
