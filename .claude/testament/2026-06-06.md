# 05:36

Rate-limit retry mission. PR delivers transient error retry with exponential backoff in `TurnRunner`.

## What was built

**`packages/claude-sdk/src/private/backoff.ts`** (new):
- `calculateBackoffDelay(attempt, random)`: `base = min(500 * 2^(attempt-1), 32_000)`; `delay = base + random() * 0.5 * base`. Jitter only ever lengthens a wait (x[1.0, 1.5]).
- `isRetryable(error)`: the single extend-point for the retryable set. Branch order: abort first (never retry); `APIConnectionError` (retry, covers `APIConnectionTimeoutError` subclass); `APIError.type` in {`rate_limit_error`, `overloaded_error`} (retry); else false.
- `defaultSleep(ms, signal)`: resolves immediately if already aborted; races `setTimeout` against the abort signal's event so it returns the instant the signal fires.
- Constants: `BASE_DELAY_MS = 500`, `MAX_DELAY_MS = 32_000`, `MAX_RETRIES = 10`.

**`packages/claude-sdk/src/private/TurnRunner.ts`** (modified):
- Added optional `sleep?` / `random?` constructor params defaulting to real implementations. `main.ts` is untouched.
- Retry loop wraps the stream+process pair only. `conversation.push` and the return stay outside. On exhaustion (11 stream calls, 10 sleeps), the last error re-throws unchanged through `QueryRunner`'s existing catch.
- Post-sleep abort check: `if (abortSignal.aborted) turnInput.abortSignal.throwIfAborted()`. Explicit and deterministic: stops the loop immediately with no further `stream()` call, regardless of what the SDK would throw.

## Error classification

In-stream errors arrive as `APIError` with `status === undefined` (HTTP 200, SSE error event). `overloaded_error` also arrives as HTTP 529 (`InternalServerError extends APIError`, `status 529`). The `.type` check covers both paths.

The SDK's own `maxRetries = 2` handles HTTP 408/409/429/>=500. In-stream errors (inside HTTP 200) are entirely outside SDK retry coverage. HTTP-level errors can double-retry (SDK 2x + our 10x) -- intentionally accepted.

## The abort path

Abort during backoff surfaces via `throwIfAborted()`, which throws `signal.reason` (a `DOMException` when `abort()` is called without a reason). Abort during `stream()` surfaces as `APIUserAbortError` from the SDK -- a different class. This divergence is intentional per the locked decision: do not claim the mechanism mirrors the SDK's internal abort, and do not manufacture `APIUserAbortError` from our layer.

The Apostle plan's Hunk 3 comment contained a false claim that `throwIfAborted()` mirrors the SDK's `makeRequest` abort mechanism (the SDK throws `APIUserAbortError`, not `throwIfAborted()`). The comment was corrected before landing.

## Test trap

`Stream.fromReadableStream` (used by all test helpers via `makeBetaStream`) silently swallows SSE error events. Only the real SSE path (`fromSSEResponse`) throws on `{type:"error"}`. The fake streamer must throw the error directly from `stream()`. Building an error-shaped SSE stream will not work.

## Live verification pending

The retry logic is verified against fakes. A real `overloaded_error` or `rate_limit_error` cannot be triggered on demand. Live verification of the retry path under real API load is pending.


# 06:21

Phase 2 — Apostle. Clock injection plan (four iterations, delivered clean in iteration 4).

## Injection shape

`systemReminders?: string[]` on `RequestBuilderOptions`. The loop in `buildRequestParams` emits one `<system-reminder>` block per entry, after the cache boundary (`cacheLastUserMessage` runs first), uncached. `TurnRunner.run()` is the assembly point: git delta first (turn-1 only, from `turnInput.systemReminder`), clock stamp last (every turn, fresh).

The `string[]` stays on `RequestBuilderOptions` only — does not propagate to `TurnInput` or `PerQueryInput`. `PerQueryInput.systemReminder` is the CLI's external contract; propagating up would require CLI changes (out of scope). `SdkQuerySummary` channel is unaffected.

## Cast type

The injection loop casts `lastUser.content as BetaContentBlockParam[]`, not `BetaTextBlockParam[]`. The content array holds mixed block types on tool-loop turns (text, tool_use, etc.). `BetaContentBlockParam` was absent from `RequestBuilder.ts` and needed to be added to the import.

## Dead branch

The defensive `typeof lastUser.content === 'string'` guard before the push is dead code: `cacheLastUserMessage` arrayifies the content before injection runs. The SDK's no-defensive-code convention applies — dead branches are debt. Push directly.

## Clock injectability at TurnRunner

`clock: Clock = Clock.systemDefaultZone()` was added as a positional constructor parameter. After the merge of #334 (which added `sleep?`/`random?`), the final constructor signature is `(streamer, processor, logger?, sleep?, random?, clock?)`. The `clock` position last means none of the existing call sites or tests need updating. See the `10:41` entry in `2026-06-07.md` for the merge resolution.
