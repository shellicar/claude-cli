## One real deviation from the plan: FakeClock needed millis() and equals()

The plan's FakeClock implementation (which I followed) was missing two abstract methods
required by `@js-joda/core` v5's `Clock`: `millis()` and `equals()`. TypeScript reported:
`Non-abstract class 'FakeClock' is missing implementations for the following members of
'Clock': 'equals', 'millis'.`

The plan noted a fallback if "the constructor is protected or the class is not open for
extension" — but the actual error was different. I added the two missing overrides:

```ts
override millis(): number { return this.#current.toEpochMilli(); }
override equals(obj: unknown): boolean { return this === obj; }
```

Both are trivially correct for a fake.

## Type-check caveat

Running `pnpm -F claude-sdk-cli type-check` fails if workspace packages haven't been
built first (module resolution errors for @shellicar/claude-core, @shellicar/claude-sdk,
etc.). Build the workspace packages first, then type-check passes clean.

Command that works:
```
pnpm --filter @shellicar/claude-core --filter @shellicar/claude-sdk --filter @shellicar/claude-sdk-tools build
pnpm --filter claude-sdk-cli type-check
```

This is pre-existing behaviour, not introduced by this change.
