Scout for cancel-tool-use mission. Branch `feature/cancel-tool-use`, commit `61d4fb6`.

## What I found

**ESC wiring** is in two hops: `AppLayout.handleKey` calls `#cancelFn()` (line 200), which is set by `runAgent.ts:50` to `() => consumerChannel.send({ type: 'cancel' })`. The `consumerChannel` subscriber in `main.ts:222–226` then calls `currentAbortController.abort()` AND `approval.handle(cancel)`. HTTP cancel works because `TurnRunner.run` passes `turnInput.abortSignal` to the Anthropic SDK stream request.

**The gap is clean.** No signal reaches tool handlers at all. `ToolHandler` type (`types.ts:13`) takes only `input`. `ToolRegistry.resolve` run closure calls `handler(parsedInput)` with no signal. The abort signal lives in the query runner turn loop but never reaches `#handleTools`.

**Exec subprocess chain** is `Exec.ts → execute.ts → execStep.ts → execCommand.ts/execPipeline.ts`. Plain `spawn` with no `signal` option. To abort: `signal.addEventListener('abort', () => child.kill())` in both `execCommand` and `execPipeline` (which has an array of children to kill). After `close`, check `signal.aborted` and reject with `AbortError`.

**Error path is safe.** `ToolRegistry.resolve` catch block (lines 80–87) catches everything, returns `handler_error`. `QueryRunner.#handleTools` turns that into `tool_result is_error: true` with the error message as content. No crash. Claude receives it.

**Elapsed time: EASY.** Add `const startMs = Date.now()` in the `run` closure before the handler call, compute elapsed in the error branch. All local to `ToolRegistry.ts:75–93`.

## Design shape (for Apostle)

The pieces fit together cleanly:

1. `ToolHandler` type gets `signal: AbortSignal` as second param.
2. `ToolResolveResult.run` signature gets `signal?: AbortSignal` as second param.
3. `ToolRegistry.resolve` run closure gets `signal` threaded in; calls `handler(parsedInput, signal)`.
4. `QueryRunner.#handleTools` gets a `signal: AbortSignal` parameter; passes it when invoking the `run` closure.
5. `QueryRunner.run` passes `input.abortController.signal` to `#handleTools` call.
6. All tool handlers update their signature to accept `signal`; only `Exec` uses it.
7. `Exec.ts` handler passes `signal` to `execute(normalised, cwd, signal)`.
8. `execute`/`execStep`/`execCommand`/`execPipeline` chain the signal; `execCommand` and `execPipeline` kill their child(ren) on abort and reject with `AbortError`.

The same `AbortController` that cancels HTTP can cancel tool handlers — no new controller needed. The signal is already created per-query in `runTurn` at `main.ts`.
