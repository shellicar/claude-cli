# History view (#179)

A separate view for looking back over the active session's sealed blocks without disturbing the live one. Switch in with F2, navigate and inspect freely (including while a query is mid-stream), switch back with F1. Restores the in-session look-back that the move to an alt buffer took away.

## How it fits together

- **View switch — no refactor.** `ViewHost` already hosts and swaps presentations keyed by `AppModeKey`; both `dispatchKey` and `renderNow` route through `#activePresentation()`. The history view slots in as one additive `historyViewState` store subscription plus a new `'history'` presentation entry. The one gotcha: the presentations `Map` must be `new Map<AppModeKey, Presentation>([...])`. TypeScript infers the value type from the first entry, so two heterogeneous presentations (`PrimaryPresentation`, `HistoryPresentation`) clash on `#view` without the explicit generic.

- **Tool-data capture (additive, Primary untouched).** A sealed tools block in `ConversationState` was only a flat string of one-line summaries — per-tool input/output was dropped when display blocks were built. It is now captured alongside, leaving Primary's rendering byte-identical:
  - The SDK publishes a new `tool_result` message (`SdkToolResult`: id + content + isError) at every site in `QueryRunner.#handleTools` that builds a client `tool_result` block (not_found, invalid_input, handler_error, success, user-rejection). Content is **post-transform** by construction — the run closure holds the already-transformed string, so a ref-swapped large output shows as the ref the model saw.
  - `AgentMessageHandler` captures input at `tool_use_input_stop`/`server_tool_use` via `ToolObject.setInput`, output at `tool_result`/`server_tool_result` via `setOutput`. `#redrawTools` calls `conversation.setLastTools(content, entries)` — same content string as before, entries beside it.
  - `setLastTools` mirrors `setLastContent`'s active-then-sealed-reverse search. This matters: tool outputs land *after* `message_usage` seals the tools block (tools run between turns), so the sealed block is updated in place.
  - The Primary-unchanged guarantee is pinned by a characterisation test: identical event sequence with and without the output event, asserting byte-identical tools-block content.

- **Navigation is a box model.** Every block is a box (header divider + content). Collapsed, each box is capped (`COLLAPSED_CONTENT_LINES = 6`, ≈8 with the header) and `...` marks more; the focused box is centred on screen with neighbours bleeding off the top/bottom edges. `[open]` grows the focused box to its full content bounded by the screen; content taller than the screen scrolls inside a fixed frame (header stays, content slides by `scrollOffset`, `...` markers appear top/bottom only when content is hidden that direction). Tools nest the same model: open a tools block to its tool list, open a tool to its input/output.
  - `HistoryView.render` does it with `#blockCard` (builds each box), `#centre` (flattens cards, slices a screen-height window so the focused midline sits on the screen midline), `#contentBox` (returns lines whole when they fit, else a scrolled window).
  - **An opened tool's i/o renders as a labelled box** (`input`/`output` label + indented value through the window helper), *not* the illustration's inline `input { ... }`. The inline form can't scroll when output is tall, which the box-model rule requires. This is the one place faithfulness to the model outranks the literal glyphs.
  - Content scroll bottom-clamp needs the rendered (wrapped) height, which only the view knows. `view/historyContent.ts` (`historyOpenLines`, `historyContentBudget`) is shared by the view (renders) and `HistoryNavHandler` (measures), so End/PgDn/Home land exactly at the box edge — no dead-zone, no over-scroll.

- **Keys are action-based.** `historyKeyMap(viewState, key) -> Action` is the pure state-aware translation; `HistoryViewState.apply(action)` mutates. Actions: `prev`/`next`, `open`/`close`, `scroll-up`/`scroll-down`, `page-up`/`page-down`, `home`/`end`. View select is direct and absolute: F1 → primary, F2 → history, reachable from any state, via `ViewSelectHandler`. F-key support was added to `KeyAction` in `claude-core`. Entry (F2 from outside history) focuses the latest block folded (`enterAtLatest`); no focus state is kept across exits — re-pressing F2 while in history leaves focus alone. Moves clamp at boundaries (no-op, no emit).

## Decisions the SC made through the arc

The navigation design was reworked from the ground up more than once: a one-line-row outline, then a fold/unfold collapsible outline, then the box model that shipped. View switching went from an enter/exit toggle to direct F1/F2 absolute select. A footer view bar (active view marked accent + underline, inactive dimmed) was added as host-level chrome. Tool output capture was settled as post-transform (a faithful mirror of what the model saw, not a file viewer). The shipped code reflects the final state of all four; the mission brief carries the full history if needed.

## Traps for the next operator

- **The plan's file:line citations are offset** (written against `f1f3b6e`) by a handful of lines throughout. Every cited construct exists — re-ground each call site before editing rather than trusting line numbers.
- **turbo caches `pnpm type-check`** (`FULL TURBO`) and won't re-run `tsc` on new source when the working tree equals the index. Trust a direct `pnpm exec tsc --noEmit` in the package, not the cached turbo result. Same for `TsDiagnostics` — it can report stale; the direct `tsc` is the gate.
- **Single-spec runs fail** in `apps/claude-sdk-cli` (documented vite workspace-resolution failure). Run `pnpm test` from the repo root.
- **Testaments ship with the PR.** `.claude/*/**/*.md` is whitelisted in this repo's `.gitignore`, so this file is on the branch.
- **tdd lesson worth keeping:** assert the produced value (the rendered rows array, exact strings) — never fold it to a boolean (`includes(...).toBe(true)`), which names no value on failure. The render-level frame tests pin the box-model format against the Navigation illustrations.
