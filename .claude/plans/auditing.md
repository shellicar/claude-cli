# Audit Log Plan

JSONL file per session. Filename is the session ID.

Each line is an SDK event with a timestamp prepended:

```
{ "timestamp": "2025-07-14T03:22:11.123Z", "type": "message_start" }
{ "timestamp": "...", "type": "message_text", "text": "Let me check..." }
{ "timestamp": "...", "type": "message_usage", "inputTokens": 3, "cacheCreationTokens": 13585, "cacheReadTokens": 8182, "outputTokens": 42, "costUsd": 0.012, "contextWindow": 200000 }
{ "timestamp": "...", "type": "tool_approval_request", "requestId": "...", "name": "Exec", "input": { ... } }
{ "timestamp": "...", "type": "done", "stopReason": "end_turn" }
```

All event types get written as they arrive. One `message_usage` per model turn
means token and cost stats accumulate naturally.

In addition to SDK events, record the conversation content by role:

```
{ "timestamp": "...", "type": "user", "subtype": "message", "text": "help me debug this" }
{ "timestamp": "...", "type": "user", "subtype": "tool_result", "name": "Exec", "input": { ... }, "output": "..." }
{ "timestamp": "...", "type": "assistant", "subtype": "text", "text": "Let me check..." }
{ "timestamp": "...", "type": "assistant", "subtype": "thinking", "text": "..." }
{ "timestamp": "...", "type": "assistant", "subtype": "tool_use", "name": "Exec", "input": { ... } }
```

User messages: what the user typed. Not system prompts, not system reminders.
Tool results: name, input, and output — what the agent actually did and saw.

One synthetic `init` line at the start:

```
{ "timestamp": "...", "type": "init", "cwd": "/path/to/repo", "model": "claude-sonnet-4-6", "tools": ["Find", "ReadFile", ...] }
```

File lives at `~/.claude/audit/<session-id>.jsonl`. Session ID is a UUID
generated by the CLI at startup.

## Gap: cache token split

`message_usage.cacheCreationTokens` is currently a single number, but the API
returns separate 1h and 5m buckets with different prices. The SDK needs to
expose both so the audit line records the correct split:

- `MessageUsage` (private): `cacheCreation1hTokens` + `cacheCreation5mTokens`
- `MessageStream`: read both from the `message_start` stream event
- `pricing.ts`: apply the two rates separately; drop the `cacheTtl` param
- `SdkMessageUsage` (public): expose both fields
- `AgentRun`: remove `cacheTtl` from the `calculateCost` call
