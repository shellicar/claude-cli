## Design (from `2026-04-14_245_design-revision.md`)

**Design B (ConversationStore)** — single stateless class, `IFileSystem` injected.

Path layout:
- Id file: `fs.cwd() + '/.claude/.sdk-conversation-id'` (project-scoped, plain UUID)
- History files: `fs.homedir() + '/.claude/conversations/<uuid>.jsonl'` (global)
- Legacy path: `fs.cwd() + '/.sdk-history.jsonl'` (migration source)

Methods: `loadId()`, `createId()`, `loadHistory(id)`, `saveHistory(id, messages)`, `migrate()`.

The store is stateless — no current-id tracking. The caller holds `let conversationId` and passes it in.

Key implementation notes from the design doc:
- `NodeFileSystem.writeFile` handles directory creation — no explicit mkdir needed
- JSONL format: one `BetaMessageParam` per line, `JSON.stringify` per message, joined by `\n`
- `migrate()` checks: if id file exists → return null immediately; if legacy `.sdk-history.jsonl` does not exist → also return null; otherwise generate UUID, copy JSONL to new history path, write id file, return UUID
