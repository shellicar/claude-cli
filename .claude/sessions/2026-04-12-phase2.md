## Decisions worth knowing

`clearConversation()` in `AppLayout` replaces `#conversationState` with a fresh instance rather than adding a `clear()` method to `ConversationState`. `ConversationState` has no clear concept — it only accumulates. Replacing the instance is the right call; don't add `clear()` to tidy it up.

`conversationId` is `let` in `main()` scope (not inside the `onNewConversation` callback) so display code can reach it when conversation id display is added to the TUI later.
